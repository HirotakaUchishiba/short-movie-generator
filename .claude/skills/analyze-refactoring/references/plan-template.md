## 記述のポイント

### 問題の記述

- ファイルパスと行番号を必ず含める
- 問題の具体的な内容を記載
- 修正方法を提案

### Phase 分割の基準

1. **Critical**: セキュリティ、データ破損リスク
2. **High**: 機能に影響、開発効率に大きく影響
3. **Medium**: 品質向上、保守性改善
4. **Low**: コードスタイル、軽微な改善
