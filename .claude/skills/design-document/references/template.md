# 設計書テンプレート

## 基本構成

```markdown
# 〇〇設計書

## 1. 背景と目的

### 現状の課題

- 課題を箇条書き

### 解決策

- 採用する技術・アプローチを明記

### 今回のスコープ

- 今回やることを明確に記載
- やらないことも明記

## 2. アーキテクチャ設計

### パッケージ構成

- ディレクトリ構造をツリー形式で

### 依存関係

- 使用するライブラリ・パッケージ

### Export 設計

- 何を export するか (型・関数・コンポーネント)

## 3. 実装設計

### 3.1 〇〇の設計

- コンポーネント・モジュールごとに
- Props / パラメータの説明
- 責務の説明

### 3.2 〇〇の設計

...

## 4. セキュリティ考慮事項

(必要な場合のみ)

## 5. テスト方針

- 単体テスト
- 統合テスト
- 手動テスト

## 6. 運用設計

(必要な場合のみ)

## 7. 実装タスク

### Phase 1: 基盤構築 (今回実装)

- [ ] 1. 〇〇のセットアップ
  - [ ] 1-1. 〇〇
  - [ ] 1-2. 〇〇
- [ ] 2. 〇〇の実装
  - [ ] 2-1. 〇〇
        ...

### Phase 2: 拡張 (将来実装)

...

## 8. リスクと対策

(必要な場合のみ)

## 9. 参考資料

- 公式ドキュメントへのリンク
```

## セクションの使い分け

### 必須セクション

1. **背景と目的**: なぜこの設計が必要か
2. **アーキテクチャ設計**: 全体構成
3. **実装設計**: 各コンポーネントの設計
4. **実装タスク**: チェックリスト形式のタスク

### オプショナルセクション

- **セキュリティ考慮事項**: 認証・認可、API 通信などがある場合
- **テスト方針**: 複雑なロジックがある場合
- **運用設計**: 本番運用に関わる場合
- **リスクと対策**: 重要な技術選定・アーキテクチャ変更の場合

## ファイル命名

`YYYY-MM-DD_feature-name.md`

例:

- `2026-05-07_full-automation-implementation-plan.md`
- `2026-05-07_full-automation-feasibility.md`
